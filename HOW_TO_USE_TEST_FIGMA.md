# 如何使用 Test Figma 功能

## 🎯 快速开始

### **步骤 1：打开应用**
启动 Demo Sample 应用，您将看到 Home Page。

### **步骤 2：点击 Test Figma**
在 Home Page 上，找到紫色的 **"Test Figma"** 按钮（在"Use Test Images"下方），点击它。

### **步骤 3：启动 ID Scanner**
在 Test Figma 页面，点击金色的 **"ID Scanner"** 按钮。

### **步骤 4：授予相机权限**
首次使用时，应用会请求相机权限。点击"允许"。

### **步骤 5：扫描 ID**
1. 将身份证放在黄色扫描框内
2. 等待相机对焦
3. 点击底部的蓝色 **"Capture"** 按钮

### **步骤 6：查看结果**
- OCR 识别完成后，自动跳转到结果页面
- 显示 **"Identity has been checked"** ✅
- 查看识别出的 ID 信息
- 点击返回按钮退出

## 📸 支持的 ID 类型

1. **香港身份证 (HKID)**
   - 格式：A123456(7)
   - 显示 ID 号码和验证状态

2. **中国居民身份证**
   - 格式：18 位数字
   - 显示 ID、出生日期、性别、验证状态

3. **护照 (Passport MRZ)**
   - 显示护照号、姓名、国籍、出生日期、性别、有效期

## 💡 使用技巧

### **获得最佳识别效果**
- ✅ 在光线充足的环境下扫描
- ✅ 将 ID 卡平整放置
- ✅ 确保 ID 完全在黄色框内
- ✅ 避免反光和阴影
- ✅ 保持相机稳定

### **如果识别失败**
1. 返回重新扫描
2. 调整光线和角度
3. 确保 ID 文字清晰可见
4. 避免手指遮挡

## 🆚 Test Figma vs Scan ID 的区别

| 特性 | Test Figma | Scan ID |
|------|-----------|---------|
| **入口** | 紫色按钮 | 黄色按钮 |
| **相机** | 实时预览 | 静态/拍照 |
| **UI** | ID Scanner 设计 | 自定义 Figma 设计 |
| **结果** | 独立结果页面 | 同页面显示 |
| **后续操作** | 仅查看 | 可处理文档 |

## ⚠️ 注意事项

1. **相机权限**
   - 首次使用需要授予相机权限
   - 如果拒绝，可在设置中重新授权

2. **隐私安全**
   - 所有 OCR 处理在本地设备完成
   - 不上传任何图片到服务器
   - 离开页面后自动清理相机资源

3. **性能考虑**
   - 实时相机预览会消耗更多电量
   - OCR 识别需要 1-3 秒
   - 不要在处理时重复点击

## 🐛 常见问题

### **Q: 相机无法启动？**
**A:** 检查应用是否有相机权限。前往系统设置 → 应用 → Demo Sample → 权限 → 相机。

### **Q: 识别速度慢？**
**A:** 确保设备性能良好，关闭其他应用释放内存。

### **Q: 识别不准确？**
**A:** 改善光线条件，确保 ID 卡文字清晰，避免模糊和反光。

### **Q: "Capture" 按钮无响应？**
**A:** 等待相机完全初始化（加载指示器消失），确保不在处理中。

### **Q: 与 Scan ID 功能冲突？**
**A:** 两个功能完全独立，互不影响，可以分别使用。

## 🎯 什么时候使用 Test Figma？

### **适合场景**
- ✅ 展示实时相机预览功能
- ✅ 测试 ID Scanner UI 设计
- ✅ 演示给客户/团队
- ✅ 体验专业扫描界面

### **不适合场景**
- ❌ 需要处理文档（生成 PDF/签名）
- ❌ 需要批量扫描
- ❌ 需要编辑识别结果

**对于这些场景，请使用 "Scan ID" 功能。**
